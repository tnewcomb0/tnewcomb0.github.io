How the Internet Works

The Internet is a very complicated thing. It is what connects the world, far more than any other medium. We all use it (all the time), but do we really have an understanding of how it works? I'll try to break it down a little and explain the many processes behind it. This is based on older information, in order to keep it a little simpler.

First, we'll start with internet addresses, and expand on our understanding from there. This address is known as an IP address (IP stands for Internet Protocol). IP addresses are in the form nnn.nnn.nnn.nnn where nnn must be a number from 0 - 255. This is quite similar to your home address, it identifies where to find you. If something (in this case information) needs to be sent to you, your IP address is the navigation marker to find you. So when sending something between 2 computers, it goes from your IP address into "the internet" (which is what we will begin to dissect next) and then into another computer, located by their IP address.

Now, we have the beginning and end of the process, but what's in the middle? An example should serve here: Let's say your IP address is 1.2.3.4 and you want to send a message to the computer 5.6.7.8. The message you want to send is "Hello computer 5.6.7.8!". The message must be translated from alphabetic text into electronic signals, transmitted over the Internet, then translated back into alphabetic text. How is this accomplished? Through the use of a protocol stack. Every computer needs one to communicate on the Internet and it is usually built into the computer's operating system (i.e. Windows, Unix, etc.). The protocol stack used on the Internet is refered to as the TCP/IP protocol stack because of the two major communication protocols used. There are 4 layers to the stack: 

      Application Protocols Layer-   Protocols specific to applications such as WWW, e-mail, FTP, etc.

      Transmission Control Protocol-  Layer TCP directs packets to a specific application on a computer using a port number.

      Internet Protocol Layer-  IP directs packets to a specific computer using an IP address.

      Hardware Layer-  Converts binary packet data to network signals and back.

Now, the rest of the intervening process is handled by the data companies that own all the expensive computing and routing equipment, which would take a very long time to go into. For now, suffice it to say that they handle the data transfer between computers. I'll try and come back to this and expand upon it, but I'm 3 hours in on this exercise.
